AquaMobile Redesign

Duration

2 weeks

Role

User Interface Designer

This was a project to improve my user interface skills. All changes were based off my frustrations and difficulties I faced as a frequent user of this app.

Major Challenges Addressed

Challenge 1: User flow

From my experience with the app, the bottom navigation did not provide easy access to most important or most used sections of the app.

Solution:

The decision to put "My Bookings" option at the most accessible spot on the bottom nav was made from the fact that I used this section every single time I opened the app in order to prepare for my upcoming lessons. I chose to put "My Availability" option in the middle because it was the second most used section.

Challenge 2: Contrast

Contrast for elements such as buttons, top navigation, banners did not meet WCAG standards.

Solution:

Switched the color scheme using th 60-30-10 rule and made sure each combination met standards.

Challenge 3: Bookings Page

Displaying lessons for different days on the same page with minimal white space resulted in wrong status completions. There was also no convenient way to message clients or get directions without copying and pasting .

Solution:

Only lessons for the selected day are displayed to decrease chances of wrong status completions. Optimized white spacing for easier comprehension of details and added tertiary buttons for convenient access to client's number and address.

Challenge 4: Availability Page

Call to actions are hard to identify due to mixture of icons and call to action with the same color scheme cramped together without labels. Makes the process of editing availability and services confusing.

Solution:

Clear call to action buttons placed in areas where it’s easy to tell what can be edited.

Other Changes

The rest of the changes were mainly converting the other pages to make everything consistent.
